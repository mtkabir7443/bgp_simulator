Justin Furuness — 1:30 PM
@everyone A few people have submitted, and their projects have undefined behavior that make the tests pass sometimes and not at other times. Make sure to have your code running in wsl, not some VS code WSL terminal, since that breaks the tests. Additionally, you can add -O3 -g0 flags to your compilation command to remove debug info and make it a release build. Sometimes undefined behavior will overwrite places that debug info lives and won't affect your program, but these flags will strip that to make it more likely to crash or fail the tests if you have undefined behavior.
